## 🔄 工作流程

```
启动应用
  ↓
检查登录状态
  ↓
未登录 → 启动 LoginActivity
  ↓
显示统一认证页面
  ↓
用户登录
  ↓
重定向到回调 URL
  ↓
提取 JSESSIONID
  ↓
返回给 MainActivity
  ↓
保存登录状态
  ↓
显示主页面
```

## 💾 数据存储

SharedPreferences（`LoginPrefs`）中存储：
- `isLoggedIn` (boolean)：登录状态
- `jsessionId` (String)：JSESSIONID 值
- `cookies` (String)：完整的 Cookie 字符串

## 🎯 核心功能

### Java 端
```java
// 启动登录
Intent intent = new Intent(this, LoginActivity.class);
loginLauncher.launch(intent);

// 提取 JSESSIONID
String jsessionId = login.extractJSessionId(cookies);
```

### JavaScript 端
```javascript
// 检查登录
var isLoggedIn = Bridge.isLogin();

// 获取 JSESSIONID
var jsessionId = Bridge.getJSessionId();

// 获取完整 Cookie
var cookies = Bridge.getCookies();
```

## 🎉 重构优势

1. **代码分离**：登录逻辑独立在 LoginActivity，主界面逻辑在 MainActivity
2. **可重用**：LoginActivity 可以在任何地方被调用
3. **状态管理**：使用 SharedPreferences 持久化登录状态
4. **结果回调**：使用 ActivityResultLauncher 优雅地处理登录结果
5. **易于维护**：各个模块职责清晰，便于后续扩展
