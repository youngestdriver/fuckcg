# 项目修复说明

## 已修复的问题

### 1. UnsatisfiedLinkError 崩溃问题

**原因**: Native 库 `libAMapSDK_Location_v6_6_0.so` 加载失败导致应用崩溃。

**修复措施**:

#### a) ChingoEncrypt.java
- ✅ 添加了 `isLibraryLoaded` 标志位来跟踪库加载状态
- ✅ 改进了异常处理,捕获 `UnsatisfiedLinkError`
- ✅ 添加了详细的日志输出
- ✅ 提供了 `isLibraryLoaded()` 方法供调用方检查

#### b) http.java
- ✅ 在调用 native 方法前检查库是否加载成功
- ✅ 添加了异常处理,防止应用崩溃
- ✅ 改进了日志输出

#### c) app/build.gradle.kts
- ✅ 添加了 `ndk.abiFilters` 配置,指定只打包 `arm64-v8a` 架构
- ✅ 添加了 `sourceSets` 配置,明确指定 jniLibs 目录
- ✅ 确保 native 库正确打包到 APK 中

## 重要说明

### 设备架构兼容性

目前项目只包含 **arm64-v8a** 架构的 `.so` 文件。这意味着:

- ✅ **支持的设备**: 64位 ARM 架构的手机 (大部分现代 Android 设备)
- ❌ **不支持**: 32位 ARM (armeabi-v7a)、x86、x86_64 架构

### 如果需要支持更多架构

如果你需要在其他架构的设备上运行,需要:

1. 获取对应架构的 `.so` 文件:
   - `armeabi-v7a/libAMapSDK_Location_v6_6_0.so` (32位 ARM)
   - `x86/libAMapSDK_Location_v6_6_0.so` (32位模拟器)
   - `x86_64/libAMapSDK_Location_v6_6_0.so` (64位模拟器)

2. 放置到对应目录:
   ```
   app/src/main/jniLibs/
   ├── arm64-v8a/
   │   └── libAMapSDK_Location_v6_6_0.so
   ├── armeabi-v7a/
   │   └── libAMapSDK_Location_v6_6_0.so
   ├── x86/
   │   └── libAMapSDK_Location_v6_6_0.so
   └── x86_64/
       └── libAMapSDK_Location_v6_6_0.so
   ```

3. 修改 `app/build.gradle.kts`:
   ```kotlin
   ndk {
       abiFilters += listOf("arm64-v8a", "armeabi-v7a", "x86", "x86_64")
   }
   ```

## 运行项目

1. **清理项目**: 
   ```powershell
   .\gradlew.bat clean
   ```

2. **构建 Debug 版本**:
   ```powershell
   .\gradlew.bat assembleDebug
   ```

3. **安装到设备**:
   ```powershell
   .\gradlew.bat installDebug
   ```

## 测试建议

1. 查看 Logcat 输出,确认库加载状态:
   - 成功: `Native library loaded successfully`
   - 失败: `Failed to load native library: ...`

2. 如果库加载失败,检查:
   - 设备架构是否为 arm64-v8a
   - APK 是否包含对应架构的 `.so` 文件
   - `.so` 文件路径是否正确
