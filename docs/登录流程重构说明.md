# 登录流程重构说明

## 重构内容

已将登录页面的打开逻辑重构到 `LoginActivity` 中，并通过 `login.startOAuthPage()` 的设计模式返回 JSESSIONID。

## 实现架构

### 1. LoginActivity（登录活动）
独立的登录 Activity，专门负责打开统一认证页面并获取 JSESSIONID：

- **全屏 WebView**：显示统一认证登录页面
- **URL 监听**：监听回调 URL `http://ggtypt.njtech.edu.cn/cgapp-server/cas/appLogin?ticket=XXXX`
- **Cookie 提取**：自动提取 Set-Cookie 并解析出 JSESSIONID
- **结果返回**：通过 `Intent` 将 JSESSIONID 返回给调用方

### 2. MainActivity（主活动）
应用的主入口，负责管理登录状态和显示主界面：

- **登录状态检查**：启动时检查 SharedPreferences 中的登录状态
- **登录流程启动**：如果未登录，启动 `LoginActivity`
- **结果处理**：接收 `LoginActivity` 返回的 JSESSIONID 并保存
- **主页面显示**：登录成功后显示应用主界面

### 3. login.java（登录工具类）
提供登录相关的工具方法：

- **extractJSessionId()**：从 cookie 字符串中提取 JSESSIONID
- **startOAuthPage()**：设计为启动 OAuth 登录流程的接口（需要从 Activity 调用）

## 工作流程

```
1. 用户启动应用（MainActivity）
   ↓
2. 检查登录状态（SharedPreferences）
   ↓
3a. 已登录 → 显示主页面
   ↓
3b. 未登录 → 启动 LoginActivity
   ↓
4. LoginActivity 加载统一认证页面
   ↓
5. 用户完成登录
   ↓
6. 页面重定向到回调 URL（带 ticket）
   ↓
7. WebViewClient 检测到回调 URL
   ↓
8. 提取 Cookie 并解析 JSESSIONID
   ↓
9. 保存到 SharedPreferences
   ↓
10. 返回 JSESSIONID 给 MainActivity
   ↓
11. MainActivity 显示成功信息和主页面
```

## 数据存储

在 SharedPreferences（键名：`"LoginPrefs"`）中存储：

- **isLoggedIn** (boolean)：登录状态
- **jsessionId** (String)：JSESSIONID 值
- **cookies** (String)：完整的 Cookie 字符串

## Bridge API

JavaScript 可以调用以下方法：

### 原有方法
- `Bridge.isLogin()` - 检查是否已登录
- `Bridge.setLogin(boolean)` - 设置登录状态
- `Bridge.getCookies()` - 获取完整的 Cookie 字符串

### 新增方法
- `Bridge.getJSessionId()` - 直接获取 JSESSIONID 值

## 使用示例

### 在 JavaScript 中调用

```javascript
// 检查登录状态
var isLoggedIn = Bridge.isLogin();
console.log('登录状态：' + isLoggedIn);

// 获取 JSESSIONID
var jsessionId = Bridge.getJSessionId();
console.log('JSESSIONID：' + jsessionId);

// 获取完整的 Cookie
var cookies = Bridge.getCookies();
console.log('完整 Cookie：' + cookies);
```

### 在 Java 中使用

```java
// 提取 JSESSIONID
String cookies = "JSESSIONID=ABC123; other=value";
String jsessionId = login.extractJSessionId(cookies);
// 返回: "ABC123"
```

1. 卸载旧版本应用（清除登录状态）
2. 安装新 APK
3. 启动应用，应该自动跳转到登录页面
4. 输入用户名密码完成登录
5. 观察日志输出：
   ```
   D/LoginActivity: Detected callback URL: http://...
   D/LoginActivity: Extracted cookies: ...
   D/LoginActivity: Extracted JSESSIONID: ...
   D/MainActivity: Login successful, JSESSIONID: ...
   ```
6. 应用会显示 Toast 提示和主页面
7. 重启应用，应该直接显示主页面（已登录状态）

## 关键代码位置

- **LoginActivity.java**：第 103-144 行 - Cookie 提取和 JSESSIONID 解析逻辑
- **MainActivity.java**：第 27-40 行 - ActivityResultLauncher 处理登录结果
- **login.java**：第 39-52 行 - extractJSessionId() 方法

## 注意事项

1. **线程安全**：所有 SharedPreferences 操作都在主线程执行
2. **数据持久化**：登录状态会保留到应用被卸载
3. **Cookie 格式**：支持标准的 Cookie 格式（分号分隔）
4. **错误处理**：如果提取不到 JSESSIONID，会记录日志但不返回结果
5. **取消登录**：用户可以按返回键取消登录流程
