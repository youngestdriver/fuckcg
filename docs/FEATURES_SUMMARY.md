# 🎉 Web 工作台完整功能总结

## 📋 本次更新内容

### 1. **三标签页导航** ✅
- 底部固定导航，包含"工作台"、"关于"、"高级设置"
- 标签页间无缝切换，保留页面状态

### 2. **可编辑 JSON 工作台** ✅
- 用户生成数据后，可直接在编辑框修改 JSON
- 支持修改任意字段（学号、姓名、速度、距离等）
- 修改后直接发送修改过的数据，无需重新生成

**工具按钮**：
- 🔧 **格式化 JSON** - 重新排版 JSON（标准化缩进）
- 📋 **复制** - 复制到剪贴板（支持新旧浏览器）

### 3. **高级设置 - 凭据管理** ✅
允许用户在前端直接查看和切换用户凭据（JWT + Secret）

**功能**：
- 🔄 **刷新凭据** - 获取当前登录用户的认证信息
- 📋 **复制** - 复制凭据分享给他人
- 🔐 **粘贴并应用** - 粘贴他人凭据立即切换用户
- 🗑️ **清除** - 清空编辑框

**应用场景**：
```
场景1：多用户测试
  用户A登录 → 刷新凭据（获取A的JWT+Secret）
            → 复制凭据
            → 粘贴给用户B → 用户B粘贴并应用
            → 用户B现在以A的身份生成和提交数据

场景2：凭据备份与恢复
  定期复制凭据保存
  若凭据丢失，粘贴备份凭据恢复访问
```

## 🔌 原生端 Bridge 接口

### 已有方法（保持不变）
| 方法 | 功能 | 返回值 |
|------|------|--------|
| `buildUploadJsonSports(sid, name)` | 生成运动数据 | JSON字符串 |
| `submitSportsData(json)` | 提交数据到服务器 | JSON字符串 |

## 🔐 安全考虑

⚠️ **JWT 和 Secret 是敏感信息**

✅ 前端已做好的防护：
- 仅在"高级设置"中显示（用户主动查看）
- 支持复制到本地剪贴板（可离线管理）
- 清除按钮方便快速清空

✅ 原生端应该做好的防护：
- [ ] 凭据加密存储（EncryptedSharedPreferences）
- [ ] 不在日志中打印完整凭据
- [ ] 应用退出时清空内存
- [ ] 可选：切换用户前验证生物识别

## ❓ 常见问题

**Q: 为什么修改 JSON 后发送，后端能收到修改后的数据？**
A: 前端改为读取 textarea 的 `.value` 属性（实时输入内容），而不是 `.textContent`（初始内容）。

**Q: 粘贴凭据后是否需要重启应用？**
A: 不需要。凭据立即应用到内存和本地存储，可继续使用工作台。

**Q: 支持同时登录多个用户吗？**
A: 目前仅支持切换（替换）凭据，不支持同时保持多个用户。若需要，可在高级设置中扩展。

**Q: 如何安全保存凭据？**
A: 使用 Android 的 `EncryptedSharedPreferences` 替代普通 SharedPreferences。详见实现指南。
